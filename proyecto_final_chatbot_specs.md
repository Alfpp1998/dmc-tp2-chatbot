# Especificaciones e Instrucciones para el Proyecto Final

**Programa:** Diploma AI Engineer – Especialización en Diseño e implementación de Chatbots  
**Módulo:** Diseño e implementación de Chatbots

---

## 1. Objetivo de la Evaluación

Desarrollar un agente conversacional inteligente (chatbot) aplicando conceptos de LLM (Large Language Models) y RAG (Retrieval-Augmented Generation). El proyecto debe demostrar integración funcional entre recuperación de información, generación de respuestas naturales y manejo básico de seguridad y alucinaciones.

---

## 2. Características del Producto Tecnológico

- Implementación de un chatbot inteligente basado en LLM + RAG.
- Capacidad para responder consultas utilizando información de documentos, APIs o bases de datos.
- Procesamiento de intenciones y entidades utilizando LangChain o Rasa.
- Integración de un modelo LLM mediante OpenAI o HuggingFace Transformers.
- Implementación de embeddings, vectorización, indexación y búsqueda semántica.
- Uso de un motor vectorial como OpenSearch.
- Manejo básico de alucinaciones y validaciones de seguridad.

---

## 3. Requisitos Técnicos

- Lenguaje principal: Python.
- Uso obligatorio de LangChain.
- Uso obligatorio de un motor de embeddings o vector database (ejemplo: OpenSearch).
- Código comentado y estructurado.
- README con instrucciones de instalación y ejecución.
- Diagrama de arquitectura del sistema.

---

## 4. Desarrollo del Proyecto

| Sesión | Entregable |
|--------|------------|
| Sesión 5 | Implementar el núcleo LLM y presentar una demo básica de conversación. |
| Sesión 6 | Desarrollar embeddings, indexación y búsqueda semántica para el módulo RAG. |
| Sesión 7 | Integrar el chatbot completo y demostrar conversaciones utilizando RAG. |

---

## 5. Entregables Finales

- Repositorio GitHub público o compartido con el docente.
- Código fuente completo del proyecto.
- Notebook funcional en Google Colab o Jupyter Notebook.
- README con pasos de ejecución.
- Documentación técnica y diagrama de arquitectura.
- Diapositivas de presentación final.

---

## 6. Estructura de las Diapositivas de Presentación

La presentación debe ser clara, técnica y orientada a demostrar el funcionamiento del chatbot desarrollado. Las diapositivas deben incluir los siguientes apartados:

1. Problema a resolver.
2. Descripción detallada del problema.
3. Justificación de la elección del problema.
4. Propuesta de solución planteada.
5. Arquitectura del sistema.
6. Herramientas y tecnologías utilizadas.
7. Conclusiones y posibles mejoras futuras.

---

## 7. Recomendaciones Finales

- Mantener una estructura clara y modular del código.
- Realizar pruebas con diferentes consultas para validar el comportamiento del chatbot.
- Documentar adecuadamente las decisiones de diseño.
- Preparar una demo funcional para la presentación final.
- Verificar que todas las dependencias estén incluidas en el proyecto.
